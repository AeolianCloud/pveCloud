# pveCloud Workflow Guardrails

本文件只定义 AI 在仓库中的工作方法，不定义项目契约。

## 核心职责

- `AGENTS.md`：仓库级读取顺序、边界和总规则
- `SKILL.md`：文档先行门禁与执行流程
- `references/*.md`：分领域实现守则
- `docs/`：项目事实、契约、架构、计划、进度

## 标准流程

1. 先看 `git status --short`，确认工作区是否有用户未提交改动。
2. 先读入口与对应 skill，不直接跳进代码。
3. 找到本次任务对应的项目文档或机器契约。
4. 先判断是契约/行为变更，还是纯视觉优化。
5. 契约/行为变更先更新文档或机器契约，再暂停等待确认。
6. 纯视觉优化可直接实现，但仍要遵守前端 guardrail。
7. 只在与任务直接相关的目录内改动，避免顺手扩散。
8. 用最小但有意义的验证命令收尾。

## AI 与文档分工

- Skill 写“AI 应该怎么做”。
- 文档写“系统是什么、接口是什么、页面应该怎样工作”。
- 迁移和配置示例写“机器最终执行什么”。

## 写作原则

- skill 用来减少 AI 重复犯错，不用来隐藏项目事实。
- 文档用来给人和代码共同消费，不用来写提示词。
- 如果某条规则只对 AI 有意义，写进 skill。
- 如果某条规则会影响实现、验收、协作或系统行为，写进文档或机器契约。
