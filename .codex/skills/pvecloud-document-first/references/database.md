# Database Guardrails

本文件定义数据库实现守则，不替代表结构契约。

## 先读什么

- `docs/server/database/design.md`
- `docs/security.md`（涉及敏感数据、审计、会话、token、上传、幂等或配置安全时）
- `server/migrations/`
- 涉及的 `docs/server/api/`、`docs/server/` 业务设计文档和相关前端页面契约

如果目录或文档很大，先定位相关表、迁移编号、API 段落和业务设计段落，不要一次性把所有迁移读进上下文。

## 契约边界

- 最终表结构以 `server/migrations/` 为准。
- 设计口径、表分组、事务边界和约束说明写在 `docs/server/database/design.md`。
- 不要只改 model 或 service 而不改迁移和设计文档。
- 用户侧功能和运营侧功能如果共享同一业务事实，数据库契约要先统一，再拆前端和 API。
- 数据库变更如果会影响接口字段、错误码、权限、页面行为、配置或部署，必须同步检查对应 owner docs。
- 涉及状态流转、资源分配、幂等、异步任务、外部副作用或补偿时，数据库设计必须写清锁定对象、唯一约束或等价持久约束、重复写入语义和失败恢复锚点；按 `references/contract-quality.md` 自检后再进入实现。

## 变更分类

以下都属于数据库契约变更，必须先更新迁移和数据库设计口径，再停下来等待维护者确认：

- 新增、删除或重命名表、字段、索引、唯一约束或外键式应用约束
- 改变字段类型、长度、默认值、空值语义、状态值或金额单位
- 改变事务边界、幂等键、唯一性规则、软删除规则或审计写入规则
- 改变敏感信息存储、脱敏展示、哈希/摘要、附件引用或文件物理路径语义
- 改变 token、验证码、密码重置、会话、短锁、限流、审计或敏感配置的持久化语义
- 新增权限种子、菜单种子、系统配置种子或其它初始化数据

纯查询优化如果新增索引，也属于数据库契约变更。

## 迁移原则

- 新增表、字段、索引、约束时，迁移和设计文档一起更新。
- 迁移文件编号必须接在当前最大编号之后，文件名说明业务目的。
- 迁移必须可重复执行，并考虑已有数据、空表和已部分初始化环境。
- 对已有表加非空字段、唯一约束或状态约束前，先明确历史数据如何回填或清理。
- 初始化或补齐权限、菜单、配置等种子数据时，必须幂等，不得重复插入。
- 不把高频查询条件塞进 JSON 字段。
- 状态值由应用层常量维护，不用数据库 enum 绑死。
- 金额、价格、配额等精确数值不得使用浮点数。
- 对外展示优先使用业务编号，不直接暴露自增 ID。
- 敏感原文不得落库；确需检索时使用受控哈希、摘要或脱敏展示字段。
- 审计日志、导出记录、会话、密码重置 token、验证码和敏感配置的留存、吊销、软删除或物理删除语义必须先写清设计口径。
- 不在迁移、种子、回填 SQL 或测试夹具里写入真实密钥、真实 token、真实密码、真实验证码或生产敏感原文。

## 注释策略

- 新增或修改迁移 SQL、种子数据、回填 SQL 或测试夹具时，必须同步补充注释，说明迁移目标、幂等判断、历史数据兼容、回填策略和重复执行语义。
- 表、字段、索引、外键、条件唯一约束、生成列、状态字段、金额字段、审计字段、敏感字段、权限种子和系统配置种子都要通过 `COMMENT` 或 SQL 注释说明设计意图。
- 对已有表加字段、索引或约束时，应注释说明为什么不会破坏历史数据，或需要维护者先处理哪些数据。
- 简单 `CREATE TABLE` 字段也不再作为免注释场景；字段含义优先通过 `COMMENT` 表达，复杂约束再用 SQL 注释补充，必要时逐行注释。

## 索引与约束

- 唯一性必须靠数据库唯一索引或等价持久约束兜底，不能只依赖 service 查询判断。
- 高频列表筛选、排序和外键式关联字段应有明确索引口径。
- 软删除表的唯一约束必须明确是否排除已删除记录。
- 条件唯一约束要考虑 MariaDB 能力边界；需要生成列或摘要列时，先写清设计口径。
- 不为了某个临时页面筛选随意加宽索引；索引应服务稳定业务查询。

## SQL 安全检查

- repository 或 service 中的查询值必须参数化，不得拼接用户输入到 SQL 字符串。
- 动态排序、动态筛选字段、动态表名、动态列名和动态 JSON path 必须来自后端白名单，不得直接使用请求参数。
- `LIKE` 查询需要明确转义 `%`、`_` 等通配符语义；如果允许模糊搜索，必须限制长度和字段范围。
- `IN` 条件必须使用 ORM/驱动支持的参数展开或安全绑定，不得拼接逗号字符串。
- 分页 `limit`、`offset`、`page`、`page_size` 必须有类型校验、上下限和默认值。
- 原始 SQL、迁移回填 SQL 和种子 SQL 不得包含真实密钥、token、密码、验证码或敏感原文。
- AI 修改查询代码时，应先搜索 `Raw`、`Exec`、`Query`、`Where`、`Order`、`Group`、`Having`、`fmt.Sprintf` 和 SQL 片段拼接。
- AI 修改批量查询、导出、列表搜索或统计代码时，应检查分页上限、筛选白名单、索引支撑、超大结果集和批量枚举风险。

## 事务与一致性

- 事务只包本地必须原子完成的步骤。
- 外部系统调用放到事务外，通过锚点、补偿或重试恢复。
- 会话、审计、任务和其它关键业务事实以 MariaDB 为准。
- Redis 只能做短 TTL 状态、限流、缓存、短锁和辅助幂等，不能替代最终业务事实。
- 先有迁移和设计口径，再有对应 API 和前端实现；不要只为某个前端页面临时加字段。
- 多表写入必须明确同事务还是异步补偿；审计、附件引用、会话吊销、权限变更等关键副作用不能悬空。
- 幂等优先依赖业务唯一键、状态检查、会话标识或任务键；不要依赖前端防重复点击。

## 回扫要求

数据库变更后至少回扫：

- `docs/server/database/design.md` 的表分组、关键规则、唯一约束和阶段边界
- 相关 `docs/server/api/` 接口字段、错误码、状态码和权限说明
- 相关 `docs/server/` 业务设计、事务边界和幂等规则
- 相关 `docs/admin/` 或 `docs/web/` 页面行为、状态展示和权限判断
- `server/config.example.yaml` 和运维文档，若数据库行为依赖配置或部署前置条件
- 计划/进度文档，若阶段范围或历史口径会误导后续实现

## 验证

- 迁移文件命名和顺序清晰。
- 设计文档与迁移一致。
- 种子数据可重复执行，不重复、不覆盖不该覆盖的用户数据。
- 相关后端测试或数据库集成测试通过；若未运行，说明原因。
- 不新增临时探测脚本或本地数据文件；确需使用时验证后清理。
