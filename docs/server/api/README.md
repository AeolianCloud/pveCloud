# Server API 文档索引

本目录维护后端 HTTP API 的当前契约。跨接口通用规则先读 `conventions.md`，具体接口按业务领域读取对应文件。

## 读取顺序

1. `conventions.md`：统一响应、错误码、鉴权、权限、幂等和限流等跨接口规则。
2. `endpoints.md`：领域文档导航和实现边界提示。
3. 任务相关的领域接口文件：只读取本次涉及的文件，避免把全部接口契约一次性读入上下文。

## 领域文件

- `platform.md`：系统检查、管理端认证会话、Dashboard、管理员、角色权限、管理员会话、系统配置。
- `web-auth-realname.md`：用户端公开配置、认证、账号资料、密码、实名和实名供应商回调。
- `admin-users-logs-files.md`：Web 用户管理、实名管理、审计日志、日志中心、文件管理。
- `product-catalog.md`：产品、套餐、价格、销售地域、系统模板、网络类型、用户端公开产品目录。
- `orders-payments-wallet.md`：用户端订单、支付、钱包，管理端钱包、支付和退款运营。
- `invoices.md`：用户端发票和管理端发票运营。
- `instances-tasks.md`：实例交付、管理端实例、异步任务、用户端实例和生命周期。
- `tickets.md`：用户端工单和管理端工单运营。
- `out-of-scope.md`：暂未开放或当前不在契约内的接口域。

## 维护规则

- 新增或修改接口时，更新对应领域文件；不要把接口字段继续堆回索引文件。
- 一个领域文件持续变大时，优先按业务子域拆分，并在本索引和 `endpoints.md` 中补链接。
- 历史计划或进度不能替代本目录下的当前接口契约。
