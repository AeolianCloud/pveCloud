开放与不开放

减少误解：已经有代码或历史文档，不等于当前阶段开放。

开放当前可围绕它继续维护
  • 管理端基础后台能力。
  • 服务器产品目录管理和公开目录接口。
  • 用户账号自助、用户实名和订单 MVP。
  • 站点配置、健康检查和基础运维文档。
不开放需要先补契约再实现
  • 支付发起、回调和支付结果确认。
  • 实例列表、实例开通、实例操作和自动交付。
  • 工单、钱包、发票、SSH 密钥管理。
  • PVE 节点、资源池、库存扣减和自动开通。
