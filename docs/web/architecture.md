# Web 前端架构

本文件描述用户端前端的目标边界、阶段范围和实现契约。

## 当前状态

- 当前准备进入 Web 基础前台阶段。
- 在 `web/` 目录真正创建并通过构建前，本文件仍是用户端前端的契约文档，不代表已有实现。
- Web 基础前台阶段只落地用户端前端壳、路由、静态页面和请求边界准备。
- 本阶段不开放用户端业务 API、用户端账号、产品、订单、支付、实例或工单。

## 定位

用户端前端面向官网、公区产品展示和用户中心。
接口边界固定为 `/api/*`。

## 独立边界

- `web/` 只调用 `/api/*`
- 不调用 `/admin-api/*`
- 不导入 `admin/` 的运行时代码
- 不创建跨前端共享运行时代码包
- 不依赖管理端权限、菜单、状态或请求封装

## 基础前台阶段范围

本阶段先落地最小可运行用户端前台：

- Home
- Products 占位页
- Pricing 占位页
- Login / Register 占位页
- User Center 占位页
- 404

这些页面只承载信息架构、静态展示、空态和未来接入点，不承载真实登录、注册、下单、支付、实例或工单流程。

## 本阶段不开放能力

- 用户端 `/api/*` 业务接口
- 用户注册、登录、会话恢复、刷新和退出
- 产品套餐真实数据读取
- 订单创建和订单列表
- 支付发起、回调和支付结果确认
- 实例列表、实例开通和实例操作
- 工单创建、回复和列表
- 用户中心真实账号资料
- 与 `admin/` 的任何运行时代码共享

## 技术倾向

当前规划沿用项目现有前端栈方向：

- Bun
- Vite
- Vue 3
- TypeScript
- Vue Router
- Pinia
- Axios

本阶段创建 `web/` 时沿用上述技术栈。后续如需引入 UI 组件库、CSS 框架或 SSR/SSG 能力，必须先更新本文档并确认。

## 目标目录口径

```text
web/src/
  api/
  assets/
  components/
  layouts/
  router/
  store/
  styles/
  views/
```

说明：

- `api/`：只放 `/api/*` 请求封装准备，本阶段不接真实业务接口
- `router/`：用户端路由定义
- `store/`：跨页面 concern，本阶段只保留必要基础状态
- `views/`：页面入口
- `layouts/`：用户端页面壳
- `components/`：仅限 `web/` 内部复用组件
- `styles/`：用户端 tokens、reset 和页面基础样式

## 路由口径

本阶段目标路由：

| 路由 | 页面 | 说明 |
| --- | --- | --- |
| `/` | Home | 用户端首页静态展示 |
| `/products` | Products | 产品能力占位，不读取真实产品数据 |
| `/pricing` | Pricing | 价格能力占位，不展示可售真实价格 |
| `/login` | Login / Register | 登录注册入口占位，不发起认证请求 |
| `/user` | User Center | 用户中心占位，不要求登录态 |
| `/:pathMatch(.*)*` | 404 | 未匹配路由 |

本阶段所有路由都是公开路由，不做登录态准入、不做权限判断、不做会话恢复。

## 请求边界

- 请求基础路径预留为 `/api/*`。
- 本阶段可以创建用户端请求封装骨架，但不得调用 `/admin-api/*`。
- 本阶段不新增用户端业务接口契约，不要求后端开放 `/api/*`。
- 后续接入真实业务接口时，必须先更新 `docs/server/api/` 和必要的数据库契约。

## 状态边界

- 本阶段不定义用户登录态、token 存储、会话刷新或退出流程。
- 本阶段不持久化用户资料、订单、实例或工单状态。
- 如需全局状态，只能用于主题、布局、导航等不影响业务契约的前端 concern。

## 契约原则

- 页面行为、状态语义、请求包装和登录流程必须以 `docs/server/api/` 和对应业务设计文档为准
- 订单金额、产品可售性、镜像、地域、支付状态等最终以后端为准
- 用户端前端负责展示与交互，不负责最终业务裁决
- 用户端展示页不能隐式承诺后台尚未开放的产品、订单、支付、实例或工单能力

## 后续阶段要求

后续开放真实用户端业务前，至少需要补齐：

- 登录态与用户中心状态模型
- 用户端 `/api/*` 接口契约
- 用户账号、产品、订单、支付、实例和工单所需数据库契约
- 订单和支付流程说明
- 工单与实例相关页面范围
- 用户端路由、权限和请求包装口径的行为细节
- 必要的管理端运营页面和权限口径

这些内容确认后，再进入对应业务实现。

## 验收口径

- `web/` 与 `admin/` 独立，不共享运行时代码
- `web/` 不调用 `/admin-api/*`
- 本阶段不新增 `/api/*` 业务接口、不新增数据库迁移
- 基础路由和 404 可访问
- 用户端静态页面在桌面和移动端可用
- `web` 构建通过
