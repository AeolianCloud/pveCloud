# Web 前端架构与规划

本文件描述用户端前端的目标边界、规划范围和未来实现契约。

## 当前状态

- 当前仓库尚未提供 `web/` 前端实现目录。
- 因此本文件当前属于“规划中的契约文档”，不是现有代码结构说明。
- 维护者、开发者和 AI 不能把本文件当作“仓库里已经存在 web 实现”的证明。

## 定位

用户端前端面向官网、公区产品展示和用户中心。
接口边界固定为 `/api/*`。

## 独立边界

- 未来 `web/` 只调用 `/api/*`
- 不调用 `/admin-api/*`
- 不导入 `admin/` 的运行时代码
- 不创建跨前端共享运行时代码包

## 计划范围

后续用户端前端预计覆盖：

- Home
- Products
- Pricing
- Login / register
- Order creation
- Payment result
- User center
- My orders
- My instances
- Tickets

## 技术倾向

当前规划沿用项目现有前端栈方向：

- Bun
- Vite
- Vue 3
- TypeScript
- Vue Router
- Pinia
- Axios

但在真正创建 `web/` 实现前，技术决策仍应以届时的实现文档和维护者确认结果为准。

## 契约原则

- 页面行为、状态语义、请求包装和登录流程必须以 `docs/server/api/` 和对应业务设计文档为准
- 订单金额、产品可售性、镜像、地域、支付状态等最终以后端为准
- 用户端前端负责展示与交互，不负责最终业务裁决

## 落地前要求

在真正创建 `web/` 实现之前，至少需要补齐：

- 页面范围与信息架构
- 登录态与用户中心状态模型
- 订单和支付流程说明
- 工单与实例相关页面范围
- 用户端路由、权限和请求包装口径

这些内容确定后，再创建 `web/` 目录和具体实现。
