# Tickets 页面契约

## 页面定位

`Tickets` 是用户端工单页面集合，用于当前登录用户提交和跟进售后、账号、产品、订单或技术支持问题。

对应路由：

- `/user/tickets` - 工单列表
- `/user/tickets/new` - 创建工单
- `/user/tickets/:ticketNo` - 工单详情

## 行为范围

- 所有工单页面都是受保护路由，未登录访问时跳转 `/login` 并携带站内 `redirect`。
- 页面进入前必须完成登录态恢复；恢复失败时清理本地登录态并回到 `/login`。
- 用户端只展示、回复、关闭当前登录用户自己的工单。
- 创建工单时可以选择分类、优先级、标题、内容、可选关联订单和可选关联实例。
- 工单可选关联当前用户自己的订单；不得通过前端传入的订单号访问或关联他人订单。
- 工单可选关联当前用户自己的实例；不得通过前端传入的实例编号访问或关联他人实例。
- 同时选择订单和实例时，二者必须属于同一来源订单；只选择实例时，后端会按实例来源订单回填关联订单。
- 用户可以回复自己的未关闭工单，也可以关闭自己的未关闭工单。
- 关闭后的工单不可继续回复。
- 工单附件只允许在当前工单详情中下载或预览，不开放公网匿名下载。

## 展示内容

### 工单列表

- 工单编号
- 标题
- 分类
- 优先级
- 状态
- 可选关联订单号
- 可选关联实例编号
- 公开标签
- 最近消息时间
- 创建时间

列表支持按状态、分类、优先级、订单号、实例编号和分页查询。

### 创建工单

- 标题
- 分类：账号问题、订单问题、产品咨询、技术支持、账务问题、其它问题
- 优先级：低、普通、高、紧急
- 可选关联订单
- 可选关联实例
- 问题描述
- 附件上传

单条消息最多 5 个附件。文件大小、允许类型和安全校验以后端为准，前端只做体验提示。

### 工单详情

- 工单基础信息
- 关联订单号（如果有）
- 关联实例编号（如果有）
- 状态、分类、优先级
- 公开标签
- 消息时间线
- 每条消息的发送方、内容、附件和发送时间
- 回复表单
- 关闭按钮

## 状态语义

- `waiting_admin`：等待客服处理。
- `waiting_user`：等待用户反馈。
- `closed`：已关闭。

用户创建工单或用户回复后展示为等待客服处理。管理员回复后展示为等待用户反馈。用户或管理员关闭后展示为已关闭。

## 关联接口

- `GET /api/tickets`
- `POST /api/tickets`
- `GET /api/tickets/{ticket_no}`
- `POST /api/tickets/{ticket_no}/messages`
- `POST /api/tickets/{ticket_no}/close`
- `GET /api/tickets/{ticket_no}/attachments/{file_id}/download`

具体字段、响应和错误码以 `docs/server/api/` 为准。

用户端只调用 `/api/*`，不得调用 `/admin-api/*`。

## 验收重点

- 未登录访问会进入 `/login`。
- 用户只能查看、回复、关闭自己的工单。
- 用户不能关联他人订单创建工单。
- 用户不能关联他人实例创建工单，也不能把自己的订单和不属于该订单的实例组合提交。
- 关闭后的工单不展示回复入口。
- 附件上传校验失败时有明确反馈。
- 附件下载或预览必须经过当前工单归属校验。
- 页面不展示支付、实例交付、PVE 节点、VMID、operation ID、用户侧 SLA 或自动处理承诺。
- 页面不展示处理人、协作者、内部备注、内部 SLA 截止时间、内部标签或后台操作历史。
