# Account Profile 页面契约

## 页面定位

`Account Profile` 是用户端账号资料编辑页，对应路由 `/user/profile`。

本页面只开放当前登录用户自己的基础资料和密码修改；个人实名由 `/user/real-name` 独立页面承载。本页面不开放钱包、发票、订单、实例、工单或 SSH 密钥管理。

## 行为范围

- `/user/profile` 是受保护路由，未登录访问时跳转 `/login` 并携带站内 `redirect`。
- 页面进入前必须完成登录态恢复；恢复失败时清理本地登录态并回到 `/login`。
- 页面展示当前用户摘要：用户名、邮箱、显示名称和状态。
- 用户名只读，不允许用户端自行修改。
- 用户可以修改邮箱和显示名称。
- 显示名称由个人资料页维护，不在注册页填写。
- 用户可以修改密码，必须提供当前密码、新密码和确认密码。
- 资料保存成功后更新本地用户摘要。
- 密码修改成功后吊销当前用户其它 active 会话，当前会话保持有效；前端提示密码已更新。

## 表单规则

- 邮箱必填，必须符合邮箱格式，且不能与其它用户重复。
- 显示名称可为空，最大长度 64。
- 修改密码时当前密码必填，新密码长度必须满足服务端密码规则。
- 新密码和确认密码必须一致。
- 邮箱重复时展示明确重复提示。
- 当前密码错误时展示明确当前密码错误提示。
- 用户被禁用时，任何资料保存或密码修改都必须失败，前端清理登录态并回到 `/login`。

## 关联接口

- `GET /api/auth/me`
- `PATCH /api/user/profile`
- `POST /api/user/password`

具体字段、响应和错误码以 `docs/server/api/` 为准。

用户端只调用 `/api/*`，不得调用 `/admin-api/*`。

## 验收重点

- 未登录访问 `/user/profile` 会进入 `/login`。
- 用户名只读，邮箱和显示名称可编辑。
- 资料保存成功后本地用户摘要同步更新。
- 修改密码需要当前密码，当前密码错误有明确提示。
- 密码修改成功后其它会话失效，当前会话保持可用。
- 禁用用户不能继续编辑资料。
