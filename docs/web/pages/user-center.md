# User Center 页面契约

## 页面定位

`User Center` 是用户中心主页，对应路由 `/user`。

用户中心是登录后用户的主页面，展示用户信息、快速入口和账户概览。

## 行为范围

- `/user` 是受保护路由，未登录访问时跳转 `/login` 并携带站内 `redirect`。
- 页面进入前必须完成登录态恢复；恢复失败时清理本地登录态并回到 `/login`。
- 用户中心展示当前用户信息、快速入口和账户概览。
- 用户中心是官网的一部分，包含官网导航菜单和页脚。

## 展示内容

### 用户信息
- 用户头像（如果可用）
- 用户名
- 邮箱
- 显示名称
- 账户状态
- 实名状态

### 快速入口
- 账号资料：`/user/profile`
- 实名认证：`/user/real-name`
- 订单管理（如果开放）
- 实例管理（如果开放）
- 工单管理（如果开放）
- 钱包管理（如果开放）

### 账户概览
- 最近登录时间
- 账户创建时间
- 实名状态
- 最近订单（如果开放）
- 最近实例（如果开放）

### 导航菜单
- 用户中心导航菜单，包含：
  - 概览：`/user`
  - 账号资料：`/user/profile`
  - 实名认证：`/user/real-name`
  - 订单管理（如果开放）
  - 实例管理（如果开放）
  - 工单管理（如果开放）
  - 钱包管理（如果开放）

## 关联接口

- `GET /api/auth/me` - 获取当前用户信息
- `GET /api/site-config` - 获取站点配置（用于实名状态等）

具体字段、响应和错误码以 `docs/server/api/` 为准。

用户端只调用 `/api/*`，不得调用 `/admin-api/*`。

## 验收重点

- 未登录访问 `/user` 会进入 `/login`。
- 用户信息正确显示：用户名、邮箱、显示名称、账户状态。
- 实名状态正确显示：未实名、核验中、已通过、已拒绝。
- 快速入口链接正确，功能正常。
- 导航菜单显示正确，当前页面高亮。
- 响应式设计适配不同屏幕尺寸。
- 页面性能良好，加载速度快。
