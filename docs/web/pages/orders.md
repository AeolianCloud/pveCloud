# Orders 页面契约

## 页面定位

`Orders` 是用户端订单列表页，对应路由 `/user/orders`。

订单 MVP 只表示购买意向和后台人工处理入口，不代表支付成功、实例开通或资源交付。

## 行为范围

- `/user/orders` 是受保护路由，未登录访问时跳转 `/login` 并携带站内 `redirect`。
- 页面进入前必须完成登录态恢复；恢复失败时清理本地登录态并回到 `/login`。
- 页面只展示当前登录用户自己的订单。
- 支持按订单状态筛选。
- 用户可以进入订单详情。
- 用户可以取消自己的 `pending` 订单。

## 展示内容

- 订单编号
- 订单状态：`pending`、`cancelled`、`closed`
- 产品名称
- 套餐名称
- 计费周期
- 网络类型
- 订单金额
- 创建时间
- 取消时间
- 关闭时间

页面不得展示支付状态、实例状态、PVE 节点、资源池、库存扣减或自动开通进度。
订单列表展示的网络类型来自订单创建时保存的快照，不从当前产品目录重新拼接。

## 关联接口

- `GET /api/orders` - 当前用户订单列表
- `POST /api/orders/{order_no}/cancel` - 当前用户取消自己的订单

具体字段、响应和错误码以 `docs/server/api/` 为准。

用户端只调用 `/api/*`，不得调用 `/admin-api/*`。

## 验收重点

- 未登录访问会进入 `/login`。
- 只展示当前用户自己的订单。
- 状态筛选和分页正常。
- 只能取消 `pending` 订单。
- 非 `pending` 订单不会展示可取消操作。
- 页面不出现支付、实例或自动交付承诺。
