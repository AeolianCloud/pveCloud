# Orders 页面契约

## 页面定位

`Orders` 是用户端订单列表页，对应路由 `/user/orders`。

订单表示购买意向、续费意向和后台处理入口，不代表在线支付成功。管理员触发交付或真实支付自动交付后，新购订单可关联用户可见实例；续费订单由后台人工确认或真实支付成功后延长实例服务期。

## 行为范围

- `/user/orders` 是受保护路由，未登录访问时跳转 `/login` 并携带站内 `redirect`。
- 页面进入前必须完成登录态恢复；恢复失败时清理本地登录态并回到 `/login`。
- 页面只展示当前登录用户自己的订单。
- 支持按订单状态筛选。
- 用户可以进入订单详情。
- 用户可以取消自己的 `pending` 订单。
- 用户可以为可支付订单进入支付页。
- 页面展示新购订单和续费订单。

## 展示内容

- 订单编号
- 订单类型：`purchase`、`renewal`
- 支付状态：`unpaid`、`paid`、`manual_confirmed`、`refunded`
- 订单状态：`pending`、`provisioning`、`fulfilled`、`error`、`cancelled`、`closed`
- 产品名称
- 套餐名称
- 计费周期
- 网络类型
- 订单金额
- 创建时间
- 取消时间
- 关闭时间
- 关联实例编号
- 最近支付摘要
- 可支付入口状态

页面不得展示商户密钥、完整回调 payload、PVE 节点、资源池、库存扣减或上游自动开通进度。实例状态通过实例页面展示。
订单列表展示的网络类型来自订单创建时保存的快照，不从当前产品目录重新拼接。

## 关联接口

- `GET /api/orders` - 当前用户订单列表
- `POST /api/orders/{order_no}/cancel` - 当前用户取消自己的订单
- `POST /api/orders/{order_no}/payments` - 当前用户为自己的订单创建支付

具体字段、响应和错误码以 `docs/server/api/` 为准。

用户端只调用 `/api/*`，不得调用 `/admin-api/*`。

## 验收重点

- 未登录访问会进入 `/login`。
- 只展示当前用户自己的订单。
- 状态筛选和分页正常。
- 只能取消 `pending` 订单。
- 非 `pending` 订单不会展示可取消操作。
- 只有后端返回可支付状态时才展示支付入口；钱包余额支付仍以后端余额和订单状态裁决为准。
- 页面不出现商户密钥、完整回调 payload 或自动交付承诺。
