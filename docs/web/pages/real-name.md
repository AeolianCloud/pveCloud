# Real Name 页面契约

## 页面定位

`Real Name` 是用户端个人实名提交和状态查看页，对应路由 `/user/real-name`。

当前只开放个人实名，通过支付宝、微信侧实名供应商或后台人工审核完成核验，不开放企业实名或已通过实名的自助修改。

## 行为范围

- `/user/real-name` 是受保护路由，未登录访问时跳转 `/login` 并携带站内 `redirect`。
- `real_name.enabled=false` 时页面展示“实名功能暂未开放”，不允许提交。
- 页面展示当前实名状态：未实名、核验中、已通过、已拒绝。
- 未实名用户可以提交真实姓名、证件类型、证件号码，并选择后台开放的实名供应商。
- 当前证件类型仅支持身份证。
- 支付宝和微信实名会从服务端获取受控跳转入口，前端不得拼接供应商 URL 或自行判定实名结果；没有可用支付宝/微信侧供应商时默认走人工审核。
- 用户从支付宝或微信侧返回后，页面调用同步接口，由服务端查询供应商结果。
- 核验中状态不允许重复提交，只允许刷新或同步当前申请结果。
- 已通过状态只展示脱敏实名信息，不允许用户端修改。
- 已拒绝状态展示拒绝原因；是否允许重新提交由后台配置决定。
- 用户端实名说明文案来自 `real_name.review_notice`。
- 当后台配置 `real_name.required_for_order=true` 时，本页是用户购买机器前必须完成的实名入口。
- 人工审核实名不展示证件图片上传，也不调用旧人工实名上传接口。

## 表单规则

- 真实姓名必填。
- 证件类型当前固定为 `id_card`。
- 证件号码必填，由服务端校验格式和唯一性。
- 实名方式必选，允许值来自服务端过滤后的 `real_name.allowed_providers`，默认值来自 `real_name.default_provider`；可用外部供应商只展示支付宝或微信，没有可用外部供应商时展示人工审核。
- 服务端没有可用外部供应商或证件摘要密钥缺失时，`real_name.enabled` 仍跟随后台实名开关，页面展示人工审核方式。
- 前端可做基础格式提示，但最终校验以后端为准。

## 关联接口

- `GET /api/site-config`
- `GET /api/user/real-name`
- `POST /api/user/real-name`
- `POST /api/user/real-name/sync`

具体字段、响应和错误码以 `docs/server/api/` 为准。

用户端只调用 `/api/*`，不得调用 `/admin-api/*`。

## 验收重点

- 未登录访问 `/user/real-name` 会进入 `/login`。
- 后台关闭实名功能时不能提交；没有可用外部供应商或证件摘要密钥缺失时默认走人工审核。
- 后台要求购买前实名时，只有已通过状态才允许继续购买机器。
- 实名方式列表和默认实名方式全部跟随后端过滤后的后台配置展示。
- 核验中和已通过状态不能重复提交，已拒绝是否可重新提交跟随后端配置和次数判断。
- 列表、详情和用户端状态中的证件号码必须脱敏。
- 前端不得把证件号码明文、供应商回调参数或敏感响应写入 URL、localStorage、console 或错误提示。
