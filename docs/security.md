# 安全基线

本文档定义 `pveCloud` 当前通用安全基线。
具体接口字段以 `docs/server/api/` 为准，数据库结构和后台系统配置种子以 `server/migrations/` 为准，运行配置项以 `server/config.example.yaml` 为准。

## 适用范围

本安全基线适用于：

- `/admin-api/*` 管理端接口
- `/api/*` 用户端接口
- `admin/` 管理端前端
- `web/` 用户端前端
- MariaDB、Redis、文件存储、SMTP、反向代理和部署配置

如果本文件与更具体的 API、数据库、前端页面、运维或配置契约冲突，以更具体的 owner docs 或机器契约为准，并同步修正本文档。

## 基本原则

- 后端是鉴权、权限、资源归属和业务状态的最终裁决点。
- 前端权限判断只用于菜单、按钮、路由可见性和体验优化，不能作为安全边界。
- 管理端只调用 `/admin-api/*`，用户端只调用 `/api/*`，两端不得共享运行时代码、token、请求封装或权限模型。
- 默认拒绝未知权限、未知状态、未知 token type、未知 issuer、未知文件类型和未声明的外部输入。
- 安全增强如果会改变接受输入、拒绝输入、返回内容、审计内容、授权方式、配置要求、存储方式或事务边界，必须先更新对应 owner docs 或机器契约，再进入实现。

## 鉴权与会话

- 管理端和用户端使用独立 JWT secret、issuer、token type 和会话表。
- 受保护接口必须校验签名、issuer、token type、过期时间、`jti` 和当前会话状态。
- 管理端受保护接口还必须校验当前管理员状态和当前数据库 RBAC。
- 用户端受保护接口必须校验当前用户状态和当前用户会话状态。
- JWT、会话快照和前端本地状态不得替代数据库中的当前会话、用户、管理员、角色和权限事实。
- 退出、会话吊销、用户禁用、管理员禁用和密码重置后，相关会话必须在后续受保护请求中失效。

## 授权与资源归属

- 管理端菜单权限控制菜单、页面路由访问和页面主数据读取。
- 管理端操作权限控制按钮、写接口、危险操作和敏感详情读取。
- 二次授权入口必须校验操作者是否有权分配目标权限或目标角色，不能只校验是否能编辑角色或账号。
- 用户端当前阶段不引入权限码；用户端接口必须基于当前登录用户做资源归属校验。
- 用户端工单和工单附件接口必须基于当前登录用户校验工单归属、可选订单归属和附件归属；不得通过路径中的工单号、订单号或文件 ID 泄露他人资源。
- 前端传入的用户 ID、管理员 ID、角色、权限、租户、状态和资源归属只能作为请求参数，不能作为可信最终事实。
- 后端更新数据时必须使用请求 DTO 白名单，不得把请求 body 直接绑定到数据库 model 或完整实体后保存。
- `status`、`role_ids`、`permission_codes`、`is_super_admin`、`reviewer_id`、`owner_id`、金额、库存、可售状态、审计字段和系统字段只能由服务端规则写入。

## 输入校验与上传

- 所有外部输入必须经过服务端校验，包括路径参数、查询参数、JSON body、表单、上传文件、分页、排序和状态值。
- 列表排序字段、筛选字段和状态值必须使用后端白名单。
- 上传文件必须校验大小、扩展名、声明 MIME、Magic Bytes 和危险扩展名。
- 原始文件名只用于展示，存储文件名必须由服务端生成。
- 文件读取和下载不得允许路径穿越，数据库只保存相对存储路径。
- CAPTCHA、密码重置 token、验证码、一次性 token 和短 TTL 状态必须有场景隔离、过期时间和失败处理。
- 批量查询、导出、批量删除、批量状态变更和搜索接口必须限制单次数量、分页上限、筛选范围和调用频率。
- 文件下载必须使用安全的 `Content-Type` 和 `Content-Disposition`；用户提供的文件名不得直接进入响应头。
- 工单附件下载和预览必须经过工单归属或管理端权限校验，不得复用公开站点 Logo 下载入口或开放通用匿名文件下载。
- SVG、HTML、可执行脚本、压缩包、超大图片和可能触发解码高资源消耗的文件类型默认不开放；确需开放时必须先补充专门契约。

## 代码层安全审查清单

AI 或维护者审查后端、前端、迁移和配置相关代码时，至少检查以下风险：

- SQL 注入：不得用字符串拼接、`fmt.Sprintf`、用户输入拼接 `WHERE`、`ORDER BY`、`LIMIT`、`IN` 或 JSON 条件；动态字段必须走白名单，动态值必须参数化。
- Mass assignment / 过度绑定：不得把请求 body 直接绑定到数据库 model 后保存；更新字段必须由 DTO 和服务端白名单显式映射。
- 资源越权：不得只依赖请求中的 `id`、`user_id`、`admin_id`、`role_ids`、`permission_codes` 或状态字段；服务端必须基于当前会话主体重新裁决。
- 路径穿越：文件读写、下载、预览、删除和压缩解压不得信任用户路径；必须限定在配置的存储根目录内。
- 命令注入：不得把用户输入拼进 shell、系统命令、脚本参数或解释器调用；确需执行外部程序时必须使用参数数组和白名单。
- SSRF：后端主动请求 URL、Webhook、回调地址、图片或远程资源时，不得直接信任用户 URL；必须限制协议、域名/IP、端口和内网地址。
- XSS：前端不得把接口或用户输入未经处理写入 `innerHTML`、`v-html` 或危险属性；后端返回富文本或文件名等展示内容时必须明确转义或白名单策略。
- CSRF 与跨站边界：如果未来引入 Cookie 凭证或跨站写操作，必须补充 CSRF、SameSite、CORS 和 Origin 校验契约；当前 Bearer Token 流程不得误配宽 CORS 暴露管理端接口。
- 开放重定向：登录、重置密码和跳转参数只能跳转合法站内路径或明确白名单域名。
- 敏感信息泄露：错误响应、日志、审计、前端调试输出、测试快照和构建产物不得包含 password、token、secret、验证码、证件号明文或真实配置。
- 安全响应头缺失：生产环境必须按部署契约检查 HSTS、CSP、`frame-ancestors` 或 `X-Frame-Options`、`X-Content-Type-Options`、`Referrer-Policy` 和缓存控制。
- 缓存泄露：受保护接口、用户资料、实名资料、审计详情、敏感配置和文件下载响应不得被共享缓存或浏览器长期缓存。
- 不安全反序列化与模板注入：不得把用户输入直接交给模板、表达式引擎、脚本解释器或反序列化为可执行对象。
- 弱随机数：token、验证码、会话 ID、文件存储名、密码重置码和摘要盐不得使用可预测随机数。
- 限流绕过：登录、注册、验证码、密码找回、token refresh、文件上传和高危管理操作必须考虑按账号、IP、会话或主体限流。
- 并发竞态：审批、发布、会话吊销、密码重置、文件删除、状态切换和未来订单支付类操作必须检查重复提交和并发状态冲突。
- 依赖与配置暴露：不得在代码、默认配置、文档示例、Docker/脚本输出或前端环境变量中写入真实密钥。
- 依赖供应链：新增或升级 Go、Bun、前端包、构建插件和脚本依赖时，必须检查 lockfile、来源、安装脚本、构建时执行行为和已知漏洞风险。

发现上述问题时，如果修复会改变接口接受/拒绝语义、返回内容、存储方式、审计内容、权限边界、配置要求或事务边界，必须先更新 owner docs 或机器契约并等待确认。

## 敏感数据

- 密码、token、secret、验证码、SMTP 凭据、数据库密码、Redis 密码、对象存储密钥和敏感配置明文不得出现在接口响应、日志、审计详情、测试输出或示例文档中。
- 密码必须只保存不可逆哈希。
- 证件号码等敏感身份信息不得明文落库；确需检索时使用受控摘要或哈希，展示时只返回脱敏值。
- 证件号码查询摘要必须使用后台敏感配置 `real_name.identity_digest_secret` 参与计算；新增实名记录不得继续使用无盐 SHA-256。历史无盐摘要只允许作为兼容旧数据的重复校验输入，不得作为新记录的唯一保护手段。
- 外部实名供应商完整响应、证件号码明文、人脸图片、证件照片、token、签名密钥和 SecretKey 不得落库、进入审计详情、业务日志或前端持久化状态；真实姓名只允许作为实名申请业务资料受控保存，不得进入审计详情、业务日志或前端持久化状态。
- MCP PVE client API 的 base URL、Bearer Token、上游原始错误和 PVE 资源标识不得返回用户端；Bearer Token 只允许写入真实 `server/config.yaml`，不得进入示例配置、日志、审计或前端环境变量。
- 异步任务 `payload`、`result`、通知记录、订单支付摘要和支付/退款流水不得保存 password、token、secret、验证码、SMTP 凭据、MCP Bearer Token、商户私钥、微信 API v3 key、签名串、支付回调敏感原文或完整上游响应；必要时只保存脱敏摘要、业务编号和错误码。
- `system_configs.is_secret=1` 的配置不得通过接口返回明文。
- 邮箱、手机号、姓名、证件号、地址、密钥片段、User-Agent 等敏感或半敏感字段在日志、审计和响应中按对应 owner docs 脱敏。
- `server/config.yaml` 视为真实配置，不作为文档来源；未经维护者明确要求不提交，维护者明确要求提交时不得在回复、日志或示例中展示、复制其中的密钥。
- 备份文件、导出文件、对象存储文件、日志归档和错误追踪系统中的敏感数据必须按生产权限和留存策略保护。
- 真实密钥疑似进入 Git、日志、构建产物、备份或对话输出时，应立即停止扩散，清理暴露位置，并轮换相关密钥。

## 错误、日志与审计

- 对外错误响应不得泄露 SQL、栈信息、内部路径、配置值、密钥、token 原文或第三方系统响应原文。
- 登录、注册、密码找回等账号枚举风险接口不得通过响应暴露账号是否存在，除非具体 API 契约明确允许。
- 用户端资源读取和写操作应避免通过 `403`、`404`、错误文案、耗时差异或列表计数泄露他人资源是否存在。
- 高危管理操作、权限变更、角色分配、会话吊销、敏感配置变更、文件上传删除和实名供应商同步必须有审计锚点。
- 实名供应商结果同步必须有审计锚点。
- 审计日志用于还原后台操作历史，不得成为敏感明文泄露渠道。
- 日志和审计必须带请求链路标识，便于排查，不应依赖用户可控字段作为唯一追踪依据。
- 审计日志不得被普通后台操作物理删除；审计查询、导出和敏感详情查看必须受权限控制并尽量脱敏。
- 审计失败策略必须明确：高危操作不得在审计完全不可用时静默成功，除非对应 owner docs 明确允许降级。

## 外部实名供应商

- 支付宝和微信侧实名供应商凭据只来自后台 `system_configs` 敏感配置，必须 `is_secret=1`；管理端查询不得回显明文，用户端公开配置不得返回这些键。
- 已开放的供应商回调入口必须验签、校验时间窗口、防重放，并按本地申请编号与供应商会话 ID 幂等处理；当前微信/腾讯云不开放异步回调，结果只通过服务端查询确认。
- 供应商查询和回调处理不得信任前端 URL 参数；前端只能触发服务端同步。
- 供应商接口请求和响应日志必须脱敏，不得记录证件号码明文、真实姓名明文、完整响应、签名串、私钥、公钥证书内容、SecretId、SecretKey 或 access token。
- 供应商返回通过结果后，服务端仍必须执行本地证件摘要唯一性检查；本地冲突时不得把当前申请标记为通过。
- 当前实名流程不提供实名附件上传或预览；历史实名附件不得被通用文件管理下载入口绕过敏感数据边界。
- 外部供应商不可用、响应无法验签或查询超时时，不能把申请置为 `approved`；应保持可恢复状态或按实名业务文档的超时策略拒绝。

## 外部支付供应商

- 支付宝和微信支付商户凭据只来自后台 `system_configs` 敏感配置，必须 `is_secret=1`；管理端查询不得回显明文，用户端接口不得返回这些键。
- 支付回调入口是公开接口，不要求 Bearer Token，但必须验签、校验金额、币种、本地交易归属、订单状态和重复通知。
- 支付创建、回调、主动同步、退款和退款同步不得信任前端传入的用户 ID、订单金额、支付状态、回调 URL 或上游交易号。
- 钱包充值、余额支付、余额退款和钱包流水不得信任前端传入的用户 ID、钱包编号、余额、变动方向、变动后余额或关联对象状态。
- 钱包余额变化必须依赖 MariaDB 事务、行锁和钱包流水幂等键；Redis、前端缓存或页面展示余额不得作为最终资金事实。
- 支付和退款接口请求、响应、审计、任务 payload/result、运行日志不得记录商户私钥、API v3 key、签名串、完整回调 payload、完整上游响应、PVE/MCP Bearer Token 或用户敏感明文。
- 渠道返回支付成功但本地状态推进失败时，必须可通过支付交易号和渠道主动查询恢复；本地已标记成功后重复回调不得重复交付、续费或退款回滚。
- 渠道退款失败或不可确认时，不得先扣回用户续费服务期；只有渠道退款成功或查询确认成功后，才能回滚本地支付生效记录。
- 生产环境支付回调基础地址和供应商通知地址必须使用 HTTPS，并在反向代理层保留原始请求体、签名相关请求头和客户端来源 IP 摘要。
- 微信支付平台公钥、平台证书和公钥 ID 支持轮换。新旧平台公钥或证书并存期间，服务端可接受仍处于有效期内的旧签名材料；过期或被撤销后必须从后台敏感配置中移除。

## 并发、幂等与事务

- 幂等不能只依赖前端禁用按钮，必须依赖业务唯一键、会话标识、状态检查、数据库唯一约束、任务键或短锁。
- 会产生外部副作用的操作必须明确事务内外边界。
- 本地数据库原子状态和审计写入应同事务完成；外部调用、邮件发送、对象存储和第三方 API 不应放入长事务。
- Redis 只能承载缓存、限流、验证码、一次性 token、短锁和短期状态，不能替代 MariaDB 中的最终业务事实。

## 前端安全边界

- `admin/` 不读取、调用或复用 `web/` 的运行时代码；`web/` 不读取、调用或复用 `admin/` 的运行时代码。
- `admin/` 不调用 `/api/*`，`web/` 不调用 `/admin-api/*`。
- 前端仅展示 owner docs 已开放的订单、支付、钱包、续费订单、实例、异步任务和工单能力；不得展示尚未开放的发票、JSAPI/openid、小程序支付、通用 PVE 运维、资源池或用户侧 SLA 承诺。管理端内部 SLA 只能作为后台处理时限展示。
- 前端遇到 `401xx` 或 HTTP 401 时按对应应用的登录态恢复流程处理；`403xx` 或 HTTP 403 不等同于未登录。
- 前端不得把权限、实名状态、产品可售性、价格、库存、订单状态或支付状态作为最终业务事实。
- 订单创建、金额计算、产品可售组合、订单状态和订单资源归属必须由后端裁决；前端不得把展示金额、订单状态或当前用户摘要作为最终事实。
- 前端不得把敏感响应写入可被浏览器、代理或第三方脚本长期保留的位置；调试日志和错误提示必须避免携带 token、密码、验证码、证件号明文和真实配置。
- 前端环境变量只允许暴露公开构建配置，不得包含服务端 secret、数据库连接、SMTP 凭据、对象存储密钥或管理端私有配置。

### 开发模式本地凭据例外

- `admin/` 和 `web/` 登录页可以在开发模式提供“记住账号密码”开发辅助入口；生产构建不得显示该入口，也不得读取或写入该开发辅助凭据。
- 开发模式下只有用户显式开启该入口并登录成功后，前端才可以把当前登录账号和密码保存到各自应用的本地浏览器存储。
- 该开发辅助凭据不得进入接口请求体之外的日志、URL、错误提示、审计、测试快照、构建产物或跨端共享状态。
- 管理端和用户端必须使用独立的本地存储 key；该凭据不得参与 token 恢复、refresh、权限判断、redirect 或服务端鉴权流程。

## 部署与配置

- 生产环境必须使用独立的管理端和用户端 JWT secret、issuer 和代理边界。
- 真实密钥默认不得提交到仓库；维护者明确要求提交真实配置时，AI 必须避免在回复、日志和示例中展示密钥内容，并在提交说明中标注包含真实配置变更。
- 反向代理必须区分 `/admin-api/*`、`/api/*`、管理端静态资源和用户端静态资源。
- 生产 CORS 必须使用明确允许来源，不得用宽泛 `*` 暴露管理端接口或带凭证请求。
- 生产响应头必须按前端和 API 边界配置 HSTS、CSP、`frame-ancestors` 或 `X-Frame-Options`、`X-Content-Type-Options`、`Referrer-Policy` 和必要的 `Cache-Control`。
- 受保护 API、敏感文件下载、审计详情和用户资料响应应避免被共享缓存保存。
- Redis、MariaDB、SMTP、文件存储和反向代理的生产依赖、启动顺序、持久化和备份要求以 `docs/operations/` 与 `server/config.example.yaml` 为准。
- 日志、备份和部署输出不得包含真实密钥、token、密码、验证码或敏感配置明文。
- 备份应限制访问权限并覆盖恢复演练；包含敏感数据的备份、日志和导出文件不得公开暴露。
- 构建、部署、安装依赖和运行脚本不得依赖不可信来源或隐式执行未经审查的远程脚本。

## 测试与验收

安全相关实现至少考虑以下回归面：

- 未登录、token 无效、token 过期、会话失效
- 低权限管理员访问高权限页面、接口和敏感详情
- 二次授权提权、自我提权和自我锁定
- 用户端访问他人资源或伪造用户 ID
- Mass assignment / 过度绑定写入受保护字段
- SQL 注入、动态排序/筛选注入和非法分页参数
- 路径穿越、非法文件名、非法下载路径和伪造文件类型
- XSS、开放重定向、SSRF 和命令注入风险入口
- CORS、CSP、安全响应头、缓存控制和前端环境变量泄露
- 账号枚举、资源存在性侧信道和批量枚举
- 重复提交、并发提交和状态冲突
- 上传非法扩展名、伪造 MIME、超大文件和路径穿越
- SVG/HTML 文件、压缩包、超大图片和响应头文件名注入
- 敏感字段响应、日志、审计和测试输出脱敏
- 依赖供应链、lockfile、安装脚本和构建产物密钥泄露
- 审计日志权限、留存、防篡改和审计失败策略
- Redis、SMTP、对象存储或外部依赖失败后的可恢复状态

修复或新增安全边界时，应优先沉淀正式回归测试；未补测试必须说明原因和剩余风险。
