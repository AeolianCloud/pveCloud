# Wallet Management 页面契约

钱包管理用于只读查看用户钱包、余额、充值记录和钱包流水。页面只调用 `/admin-api/*`。

## 页面范围

- 钱包账户列表
- 钱包详情
- 钱包流水列表
- 钱包充值记录列表
- 用户摘要和关联对象跳转

本页面 v1 不支持管理端人工加款、扣款、冻结、解冻、提现、余额转账、退款入钱包、发票开具或后台创建充值。真实支付退款仍在支付管理页处理；余额支付退款由支付退款流程退回钱包余额。钱包充值不开具发票，发票 v1 只基于已支付订单申请。

## 路由与权限

- 路由：`/wallets`
- 菜单权限：`page.wallets`
- 查看：`wallet:view`

## 页面结构

钱包管理包含钱包列表、钱包详情、流水列表和充值列表，按复杂管理页结构实现：

```text
admin/src/views/wallets/
  index.vue
  types.ts
  components/
```

## 行为约束

- 钱包列表支持按钱包编号、用户关键字和状态筛选。
- 钱包流水支持按钱包编号、用户关键字、方向、类型、关联编号和创建时间范围筛选。
- 充值列表支持按钱包编号、用户关键字、供应商、方式、状态、充值编号和创建时间范围筛选。
- 详情展示钱包余额、累计充值、累计消费、累计退回钱包、最近流水和最近充值。
- 页面不得展示渠道完整响应、签名串、商户密钥、用户敏感资料明文或内部锁信息。
- 页面不得出现调账、提现、冻结、解冻、转账或退款入钱包按钮；后端也不提供对应接口。

## 关联接口

接口字段和响应结构以 `docs/server/api/orders-payments-wallet.md` 为准。

- `GET /admin-api/wallets`
- `GET /admin-api/wallets/{wallet_no}`
- `GET /admin-api/wallet-ledger`
- `GET /admin-api/wallet-recharges`

## 验收重点

- 侧栏菜单来自后端 `menus`，本地路由只作为页面组件白名单。
- 无权限访问 `/wallets` 时展示管理端 403 反馈。
- 页面只读，没有任何会改变钱包余额的操作入口。
- 敏感配置、完整回调 payload 和完整上游响应不在页面、控制台日志或错误提示中出现。
- 钱包管理不调用 `/api/*`，不复用用户端运行时代码。
