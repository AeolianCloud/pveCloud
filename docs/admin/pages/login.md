# Login 页面契约

## 页面定位

`Login` 是管理端登录入口。

它不挂载在后台壳内，不出现在侧栏菜单中。

## 行为范围

- 首次进入登录页时获取验证码。
- 登录成功后写入 token、本地登录态和最小管理员摘要。
- 登录错误留在登录页处理。
- 已登录用户访问登录页时，可按路由重定向规则进入默认受保护页面。

## 开发模式记住账号密码

- 管理端登录页仅在开发模式显示“记住账号密码”按钮或勾选项，生产构建不得显示该入口，也不得读取或写入该开发辅助凭据。
- 开发模式下用户开启“记住账号密码”后，登录成功才将当前登录账号和密码保存到管理端本地浏览器存储；关闭该选项或清空登录表单时应清理已保存的开发辅助凭据。
- 该能力只用于本地开发提效，不参与登录态恢复、token refresh、权限判断、审计或任何服务端流程。
- 管理端和用户端使用独立的本地存储 key，不共享账号、密码、token 或其它运行时状态。

## 会话恢复

- 首次进入或刷新后，如果本地存在 token，前端优先调用 `GET /admin-api/auth/me` 恢复登录态。
- `GET /admin-api/dashboard` 只负责首页数据，不承担登录态恢复职责。
- `POST /admin-api/auth/logout` 无论成功失败都要清理本地状态。
- `POST /admin-api/auth/refresh` 失败按登录过期处理。

## 关联接口

以 `docs/server/api/` 中认证相关契约为准。

管理端只调用 `/admin-api/*`。

## 权限

登录页本身不要求页面权限。

登录成功后的可访问页面由路由 `meta.permission`、权限 store 和后端返回的权限码共同决定。
