# Real Name Management 页面契约

## 页面定位

`Real Name Management` 用于后台查看和同步用户端个人实名申请。

当前承载个人实名申请查看、支付宝/微信侧供应商核验结果查看与同步。不承载企业实名、后台人工审核、订单拦截记录或证件库管理；供应商密钥配置在系统设置的“实名设置”分组维护。

## 路由结构

页面：

- 实名管理：`/web/real-names`

`实名管理` 作为管理端侧栏中的独立菜单项展示，不挂在 `System Settings` 下。

## 页面职责

- 分页展示用户实名申请。
- 支持按用户关键词、实名状态、证件类型、实名供应商、供应商状态和提交时间筛选。
- 查看实名申请详情，展示用户摘要、真实姓名、脱敏证件号码、证件类型、实名供应商、供应商会话摘要、供应商状态、供应商结果码、提交时间、核验完成时间、申请状态、供应商结果和失败原因。
- 对外部供应商申请执行结果同步，由服务端查询支付宝或微信侧供应商结果。
- 不支持后台代用户新增实名申请。
- 不支持后台直接修改用户实名姓名或证件号码。
- 不支持后台人工通过或拒绝实名申请。
- 不支持删除实名申请。

## 权限建议

- 页面入口：`page.real-name-management`
- 列表和详情资源：`real-name:view` 或 `real-name:*`
- 同步供应商结果：`real-name:sync` 或 `real-name:*`

## 关联接口

- `GET /admin-api/real-name-applications`
- `GET /admin-api/real-name-applications/{id}`
- `POST /admin-api/real-name-applications/{id}/sync`

具体字段、响应和错误码以 `docs/server/api/` 为准。

## 验收重点

- 管理端页面只调用 `/admin-api/*`。
- 侧栏入口由服务端 `menus` 下发。
- 页面按钮显隐使用统一权限能力判断。
- 外部供应商申请通过回调或同步结果完成状态流转，不提供人工审核入口。
- 证件号码在列表和详情中默认脱敏展示。
